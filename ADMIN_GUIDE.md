# 📚 社团管理员指南

本文档为ISA社团管理员提供日常管理指导。

## 🎯 管理员职责

1. **审核成员信息**：及时审核新提交的成员登记Issue
2. **组织活动**：策划和组织社团技术活动
3. **维护仓库**：管理仓库权限、标签、Projects等
4. **答疑解惑**：帮助新成员熟悉GitHub和社团流程

## 🏷️ 标签管理

### 创建必要的标签

进入仓库的 `Issues` > `Labels` 页面，创建以下标签：

| 标签名称 | 颜色 | 说明 |
|---------|------|------|
| 成员登记 | `#0075ca` | 成员信息登记 |
| 已确认 | `#0e8a16` | 已确认的成员 |
| 待审核 | `#fbca04` | 待审核的信息 |
| 前端开发 | `#d4c5f9` | 前端技术方向 |
| 后端开发 | `#c5def5` | 后端技术方向 |
| 人工智能 | `#f9d0c4` | AI/ML方向 |
| 数据分析 | `#bfdadc` | 数据分析方向 |
| 网络安全 | `#d93f0b` | 网络安全方向 |
| 嵌入式 | `#fef2c0` | 嵌入式开发 |

### 使用标签的建议

- 每个成员登记Issue至少应有 `成员登记` 和一个技术方向标签
- 审核通过后，将 `待审核` 改为 `已确认`
- 可以根据需要添加更多自定义标签

## ✅ 审核成员流程

1. **检查信息完整性**
   - 姓名、班级、学号是否填写
   - 联系方式是否有效
   - 技术方向是否明确

2. **添加评论确认**
   ```markdown
   ✅ 已确认！欢迎加入ISA社团！

   你的信息已录入，这是你的成员编号：#[Issue编号]

   请加入我们的社团群：[群号/二维码]
   ```

3. **更新标签**
   - 移除 `待审核` 标签
   - 添加 `已确认` 标签
   - 添加对应的技术方向标签

4. **关闭Issue（可选）**
   - 如果只用于信息收集，可以关闭已确认的Issue
   - 如果要保持开放以便成员更新信息，可以保持打开状态

## 📊 使用Projects看板

### 创建成员管理看板

1. 点击仓库的 `Projects` 标签
2. 点击 `New project`
3. 选择 `Board` 或 `Table` 模板
4. 设置以下列/状态：
   - 📝 待审核
   - ✅ 已确认
   - 🎯 技术小组（可按方向分组）

### 看板使用技巧

- 将新提交的Issue自动添加到"待审核"列
- 审核通过后移动到"已确认"列
- 可以按技术方向创建不同的视图
- 使用筛选功能快速找到特定成员

## 🔒 隐私保护要点

### 仓库权限设置

1. **设置为私有仓库**（强烈推荐）
   - Settings > Danger Zone > Change visibility > Make private

2. **添加协作者**
   - Settings > Collaborators
   - 只添加已确认的社团成员
   - 设置适当的权限级别（Read/Write/Admin）

3. **定期审查权限**
   - 每学期开始和结束时检查协作者列表
   - 移除已毕业或退出的成员

### 信息安全注意事项

- ⚠️ 不要将包含敏感信息的仓库设为公开
- ⚠️ 提醒成员不要在Issue中泄露密码、身份证号等敏感信息
- ⚠️ 导出成员数据时注意保密
- ✅ 定期备份重要信息

## 🤖 GitHub Actions自动化

本仓库已配置自动化工作流：

- **自动欢迎**：新成员提交Issue后自动发送欢迎消息
- **自动标签**：根据内容自动添加技术方向标签
- **提醒通知**：可以配置定期提醒和通知

### 查看工作流运行状态

- 点击仓库的 `Actions` 标签
- 查看最近的工作流运行记录
- 如有失败，查看日志排查问题

## 📈 数据统计与分析

### 统计成员信息

可以使用以下方式统计：

1. **通过Issues标签**
   - 查看各技术方向的成员数量
   - 筛选特定条件的成员

2. **通过Projects看板**
   - 使用Table视图导出数据
   - 制作统计图表

3. **使用GitHub CLI**
   ```bash
   gh issue list --label "成员登记" --state all
   ```

## 🆘 常见问题处理

### 成员信息填写不规范

在Issue下评论：
```markdown
你好，请补充完整以下信息：
- [ ] 班级
- [ ] 学号
- [ ] 技术方向

补充完成后请回复，谢谢！
```

### 成员要求修改信息

- 让成员在原Issue下评论说明修改内容
- 或者让成员重新提交Issue，关闭旧Issue

### 成员无法访问仓库

1. 确认已发送邀请
2. 检查成员是否接受了邀请
3. 确认用户名是否正确
