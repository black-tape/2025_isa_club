# 🤝 贡献指南

欢迎为ISA社团项目做出贡献！本文档将帮助你了解如何参与社团项目。

## 📌 参与方式

### 1. 提交成员信息

- 使用Issue模板提交你的个人信息
- 填写完整、准确的信息
- 查看README了解详细步骤

### 2. 参与讨论

- 在 [Discussions](../../discussions) 区发起或参与话题讨论
- 分享你的学习经验和技术见解
- 提出改进建议

### 3. 完善文档

如果你发现文档有错误或可以改进的地方：

1. Fork本仓库
2. 创建新分支：`git checkout -b improve-docs`
3. 修改文档
4. 提交Pull Request

### 4. 贡献代码

如果你想为社团项目贡献代码：

1. 查看 [Issues](../../issues) 寻找任务
2. 评论表达你的兴趣
3. Fork仓库并创建分支
4. 编写代码和测试
5. 提交Pull Request

## 📝 Issue提交规范

### Bug报告

使用以下模板：

```markdown
**问题描述**
简要描述遇到的问题

**复现步骤**
1. 访问...
2. 点击...
3. 发现...

**期望行为**
描述期望的正常行为

**截图**
如果可能，提供截图

**环境信息**
- 操作系统：
- 浏览器：
```

### 功能建议

```markdown
**功能描述**
简要描述建议的新功能

**使用场景**
这个功能将解决什么问题？

**实现思路**（可选）
你认为可以如何实现
```

## 🔀 Pull Request规范

### 提交信息格式

使用清晰的提交信息：

- `feat: 添加新功能`
- `fix: 修复问题`
- `docs: 更新文档`
- `style: 代码格式调整`
- `refactor: 代码重构`
- `test: 添加测试`

### PR描述模板

```markdown
## 变更内容
简要描述这个PR做了什么

## 相关Issue
Closes #[issue编号]

## 测试
- [ ] 本地测试通过
- [ ] 已添加必要的文档

## 截图（如适用）
```

## 💻 开发环境设置

如果需要在本地开发：

```bash
# 克隆仓库
git clone https://github.com/yourusername/2025_isa_club.git

# 进入目录
cd 2025_isa_club

# 创建新分支
git checkout -b feature/your-feature-name

# 开发完成后提交
git add .
git commit -m "feat: your feature description"
git push origin feature/your-feature-name
```

## 🎯 代码规范

- 保持代码简洁易读
- 添加必要的注释
- 遵循项目现有的代码风格
- 确保没有语法错误

## 📚 文档规范

- 使用清晰的中文表达
- 提供代码示例
- 添加适当的标题和分节
- 使用Markdown格式

## ✅ 行为准则

### 我们的承诺

为了营造开放和友好的环境，我们承诺：

- 尊重不同的观点和经验
- 优雅地接受建设性批评
- 关注对社团最有利的事情
- 对其他成员保持同理心

### 不当行为

以下行为被视为不当：

- 使用性化的语言或图像
- 侮辱性/贬损性评论和人身攻击
- 公开或私下骚扰
- 未经许可发布他人私人信息
- 其他不道德或不专业的行为
