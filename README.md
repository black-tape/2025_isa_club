# 🎓 2025 ISA 社团成员信息管理系统

欢迎大家加入这个小家庭！这是一个基于GitHub的社团成员信息管理项目。

## 📋 项目简介

本项目用于收集和管理2025年ISA社团成员信息，帮助社团更好地组织活动、了解成员技术方向，促进成员之间的交流与合作。

## 🚀 如何提交你的信息

### 方式一：通过Issue模板提交（推荐）

1. **点击仓库顶部的 `Issues` 标签**

2. **点击绿色的 `New issue` 按钮**

3. **选择 "社团成员登记" 模板**

4. **填写你的个人信息**
   - 姓名
   - 班级
   - 学号
   - 联系方式
   - 技术兴趣和学习方向
   - 加入社团的期望

5. **点击 `Submit new issue` 提交**

6. **等待管理员确认** - 我们会尽快回复并添加标签

### 方式二：查看示例

查看已有的成员登记Issue，了解格式要求。

## 👥 成员列表

可以通过以下方式查看所有成员信息：

- 点击 `Issues` 标签，筛选 `成员登记` 标签
- 查看 [Projects](../../projects) 看板（如果已创建）

## 🛡️ 隐私保护

**重要提醒：**

- ⚠️ 建议将本仓库设置为 **私有仓库**（Private），仅邀请社团成员协作
- 🔒 不要在公开仓库中提交敏感个人信息
- ✅ 所有提交的信息仅用于社团内部管理和活动组织

### 如何设置为私有仓库？

1. 进入仓库的 `Settings` 页面
2. 滚动到最底部的 `Danger Zone`
3. 点击 `Change visibility`
4. 选择 `Make private`

### 如何添加社团成员为协作者？

1. 进入仓库的 `Settings` > `Collaborators`
2. 点击 `Add people`
3. 输入成员的GitHub用户名或邮箱
4. 发送邀请

## 📊 社团管理功能

### 使用标签（Labels）管理

管理员可以为Issue添加以下标签：

- `成员登记` - 成员信息登记
- `已确认` - 已确认的成员
- `待审核` - 待审核的信息
- `前端开发`、`后端开发`、`AI` 等技术方向标签

### 使用Projects看板

可以创建GitHub Projects来可视化管理成员信息：

1. 点击仓库的 `Projects` 标签
2. 创建新的Project（选择Board或Table视图）
3. 将成员登记Issue添加到看板中
4. 按技术方向、学习进度等分类

## 📖 新手指南

3. 完成邮箱验证
4. 告诉管理员你的GitHub用户名，等待添加为协作者
